# SphereCord — Plugin & Feature Ideas

> A grounded, scored catalogue for **SphereCord**, the Discord desktop client of **Project Colony**. 55 curated ideas (distilled from 76 raw across 8 lenses), plus a completeness pass. Generated by a multi-agent brainstorm grounded in the real SphereCord architecture.

## How to read this

Each idea is tagged by **surface** — where it ships — which decides whether it survives Equicord's independent auto-updates:

| Surface | What it means | Update-proof? |
|---|---|---|
| `Shell patch` | A renderer patch bundled in SphereCord (`src/renderer/patches/`) | ✅ Yes — ships in the shell |
| `Native` | Electron main process (tray, hotkeys, IPC, sockets) | ✅ Yes — shell-owned |
| `Colony` | Glue to the wider ecosystem (shared config, launcher, presence) | ✅ Yes — shell-owned |
| `Equicord` | A real Vencord/Equicord plugin | ⚠️ Needs a fork/upstream — **none of these require it** |

`effort` S/M/L · `impact` 1–5 · `score` = feasibility + impact + Colony-fit + novelty (0–100). Legend: 🚩 flagship · ⚡ quick win · ➕ gap-filler.

## ⚡ Quick wins — small effort, high value

- **Runtime 8-way accent override** — Your Colony accent (amber, violet, whatever) finally reaches Discord's brand color. _(score 90, `Shell patch`)_
- **Keyboard theme & accent switcher (Ctrl+Shift+T)** — Fuzzy-type 'gruvbox', hit enter — instant retheme, no scrolling the grid. _(score 85, `Shell patch`)_
- **Account-age & risk dot in member list / message headers** — Spot throwaway raid accounts at a glance — a red dot means 'created yesterday' before they type. _(score 82, `Shell patch`)_
- **Default-deny web-permission & sensor hardening** — Discord-in-a-browser can ask for USB and location. Say no, everywhere. _(score 78, `Native`)_
- **WebRTC IP-leak guard + fingerprint consistency** — Voice chat shouldn't hand strangers your home IP. _(score 76, `Native`)_
- **Colony-palette-aware splash screen** — The loading screen wears whatever Colony theme you're running. _(score 71, `Native`)_
- **SphereCord Rich Presence (passive Colony advertising)** — Show the world you're on the Colony Discord client, not vanilla Discord. _(score 74, `Shell patch`)_

## 🚩 Flagship — signature features

- **Live Colony theme + accent sync watcher** — Change your ecosystem theme once — Discord repaints instantly. _(score 96, `Colony`)_
- **Cross-app Rich Presence / Colony Presence Hub** — "Listening to <track> via Grape" — the whole Colony ecosystem in one Discord status. _(score 83, `Colony`)_
- **Colony Command Palette (Ctrl+K)** — One shortcut to rule Discord nav, SphereCord settings, and the whole Colony ecosystem. _(score 87, `Shell patch`)_
- **Colony 'Private Mode' / Focus master switch** — Flip one Colony toggle; every app — including Discord — hardens and goes DND at once. _(score 84, `Colony`)_
- **Client-side raid radar with native escalation** — Discord's raid detection is server-side and slow; yours fires on the desktop the instant joins spike. _(score 80, `Shell patch`)_

## The catalogue

### Colony ecosystem integration

- **Live Colony theme + accent sync watcher** 🚩 — `Colony` · effort M · impact ●●●●● · **96/100**
  Change your ecosystem theme once — Discord repaints instantly.
  The single highest-value seam named across four separate lenses (colony-integration, theming, native-desktop). A native module fs.watch's ~/.config/colony/config.json (XDG path Colony already uses via config.rs), reads active palette + 8-way accent, maps 1:1 onto the matching static/colonyThemes/colony-*.css (both generated from the same theme.rs), and pushes over a new COLONY_THEME_CHANGED IpcEvent to a renderer patch that swaps a single <style id='colony-live-theme'> or flips Vencord's enabled theme. Gated by a syncColonyTheme Settings toggle so manual picks still work. This is THE feature that makes SphereCord feel like a first-class Colony window rather than a themed Discord fork; reuses the existing 1:1 mapping and installColonyThemes registration pattern. Merged from 4 near-duplicate raw entries.
  _Watch out:_ risk: couples to Colony's JSON schema — needs defensive/versioned parse; drive Vencord's theme store API rather than racing its settings.json writer; resolve manual-pick-vs-sync via last-writer-wins + toggle.

- **Colony 'Private Mode' / Focus master switch** 🚩 — `Colony` · effort M · impact ●●●●○ · **84/100**
  Flip one Colony toggle; every app — including Discord — hardens and goes DND at once.
  Merges the 'Private Mode master switch', 'Colony Focus Sync', and 'Colony-synced focus/auto-DND' raw ideas. A native fs.watch (same watcher pattern as theme-sync) on ~/.config/colony reads a shared privacy/focus flag; when set it simultaneously flips Discord to DND, suppresses badge/taskbar flash, and can enable the telemetry firewall + silent-signals via IPC + existing settings listeners. One ecosystem switch, many effects — establishes a cross-app focus/privacy contract other Colony apps honor. Depends on Colony persisting the flag (schema coordination upstream).
  _Watch out:_ Shares the theme-sync watcher infrastructure; sequence after it.

- **Cross-app Rich Presence / Colony Presence Hub** 🚩 — `Colony` · effort L · impact ●●●●● · **83/100**
  "Listening to <track> via Grape" — the whole Colony ecosystem in one Discord status.
  Merges 'Cross-app Rich Presence hub', 'Colony Presence Hub', and the MPRIS-bridge idea. Extends SphereCord's ALREADY-WIRED arRPC pipeline (main sends IpcEvents.ARRPC_ACTIVITY, renderer consumes via VesktopNative.arrpc.onActivity) to watch a Colony activity file / launcher process list under ~/.config/colony/state and synthesize a unified Rich Presence under one registered 'Project Colony' app id — swapping details/state/large_image per active app (Grape now-playing, Spotter GPU load, etc). arRPC already fetches external image URLs so per-app art needs no re-upload. Flagship ecosystem-glue feature exploiting desktop process knowledge Vencord can't reach. Main cost: registering the Discord app id + asset keys and a stable cross-app activity contract.
  _Watch out:_ Grape Listen-Along (type-2 music activity + join secret + colony:// deep link) is a natural extension once this ships.

- **spherecord:// / colony:// deep-link handoff** — `Native` · effort M · impact ●●●○○ · **78/100**
  Launcher click opens SphereCord straight to a server, DM, or voice channel.
  Merges the two deep-link entries. Register a spherecord:// (and/or colony://) scheme via setAsDefaultProtocolClient alongside the existing equibop:// handler in vesktopProtocol.ts, route spherecord://channels/<guild>/<channel> or /dm/<user> through the single-instance argv handoff into the existing RPC_DEEP_LINK / NAVIGATE_SETTINGS IpcCommands nav plumbing. Gives the Colony launcher a first-class way to open and navigate Discord like it launches other Colony apps. Protocol registration + argv forwarding are main-process only.
  _Watch out:_ risk: validate untrusted deep links before navigating; scheme registration can collide with another app; route→Discord-nav mapping needs a stable renderer hook.

- **Tray 'Colony hub' submenu (launch siblings + status + theme)** — `Colony` · effort M · impact ●●●○○ · **72/100**
  Right-click the tray: launch Grape, set status, swap Colony theme.
  Merges 'Colony ecosystem launcher menu' + 'Tray Colony hub submenu'. Extends tray.ts (StatusNotifierItem via libvesktop, already builds custom menus) with a submenu that lists installed Colony apps (detected from .desktop entries / Start Menu the way the colony launcher does) and launches them, offers a Discord status setter over IPC, and quick-switches the 52 bundled themes. Native tray app-launch can't be done from a renderer plugin. Positions the Discord client as an ecosystem portal.
  _Watch out:_ risk: sibling-app detection depends on the launcher registry/.desktop format not yet a stable public contract; launching external binaries needs a trust/allowlist check.

- **Colony member identity badge / flair** — `Colony` · effort M · impact ●●●○○ · **66/100**
  Sign into Colony once, wear the badge in Discord.
  Merges 'Colony member identity badge' + 'Colony Member Flair'. A native module reads Colony's GitHub-OAuth username (from ~/.config/colony state written by oauth.rs — public username only, NEVER the keychain token), exposes it over a new IPC channel, and a renderer patch renders a small Colony chip in the SphereCord Settings tab and beside usernames in chat/profiles (webpack patch on the username/message-header module), tinted to the live accent (--brand-500). Reuses Colony's existing identity so one ecosystem sign-in yields visible membership — no separate login.
  _Watch out:_ risk: keychain read must stay opt-in and public-username-only; client-side render means others only see it if they also run SphereCord; account-panel DOM is a moving target.

### Theming power-tools

- **Runtime 8-way accent override** ⚡ — `Shell patch` · effort S · impact ●●●●○ · **90/100**
  Your Colony accent (amber, violet, whatever) finally reaches Discord's brand color.
  Merges the three accent-override entries. Today CSS_MAP sends every --brand-*/--text-link/--control-brand-foreground to the palette's accent_blue, so Colony's 8-way accent (red/orange/yellow/green/blue/indigo/violet/amber + auto) is dropped — a gap grounding explicitly flags. A tiny renderer patch reads the current accent key (from the same native config watcher via VesktopNative, or a Settings store field) and injects a small <style id='colony-accent'> overriding --brand-experiment/--brand-500/--brand-560/--control-brand-foreground/--text-link/--text-brand to the chosen hex, porting Colony's accent_key_to_color() as a hardcoded JS lookup. Instant, layers on any active theme, ships in the shell. S-effort + closes a named gap = top quick win.
  _Watch out:_ risk: cover the full brand-shade ramp (--brand-345..660) or a few spots stay off-accent; vendor/pin the 8 hexes so a Colony accent change doesn't silently diverge.

- **Keyboard theme & accent switcher (Ctrl+Shift+T)** ⚡ — `Shell patch` · effort S · impact ●●●●○ · **85/100**
  Fuzzy-type 'gruvbox', hit enter — instant retheme, no scrolling the grid.
  A shell-renderer overlay that fuzzy-searches the 52 bundled themes (launcher order + EN labels already known from the generateColonyThemes naming scheme), toggles the matching theme in Vencord's enabled-themes settings instantly, plus a row to flip the 8-way accent. Complements colonyThemeHeaders.ts by making the same set keyboard-driven. S-effort, showcases Colony's signature theming as launcher-style UX inside Discord. Full accent support depends on the accent-override idea above.
  _Watch out:_ Pairs with the accent-override patch; could share the same overlay component as the command palette.

- **Auto light/dark pairing that follows the OS** — `Native` · effort M · impact ●●●●○ · **80/100**
  Sunset flips your whole client from Gruvbox Light to Gruvbox Dark.
  A native module subscribes to Electron nativeTheme 'updated' (OS/portal color-scheme signal) and, given a chosen Colony family, swaps between that family's light and dark colony-*.css variants (many families ship as pairs in theme.rs). Reuses the same enabled-theme flip + IPC bridge as the sync watcher; a Settings toggle picks 'follow system' vs a pinned variant. Leans on a desktop power (OS scheme events) a pure Vencord plugin can't reach + Colony's paired palettes.
  _Watch out:_ risk: needs a curated light/dark pairing table with fallback for single-variant families; nativeTheme may not fire without a desktop portal on some Linux setups — dconf/gsettings fallback.

- **In-client Colony launcher-style Themes picker** — `Shell patch` · effort M · impact ●●●●○ · **78/100**
  A first-party theme grid that looks like the Colony launcher, not Discord's flat list.
  Merges the two launcher-style-picker entries. A React component in the existing SphereCord Settings tab renders the 52 themes in Colony launcher order, family-grouped (labels already parsed by generateColonyThemes.mts), each card a live bg/text/accent swatch parsed from the colony-*.css, plus a row of 8 accent swatches and a search/filter box; one-click enable writes Vencord's enabled-theme entry + accent override. Reads theme colors over IPC from DATA_DIR/themes. Complements (not replaces) Equicord's flat grid — value hinges on the swatch preview + Colony ordering being clearly nicer.
  _Watch out:_ risk: keep enabled-state single-sourced with the native Themes tab so two pickers don't disagree; stay in sync as installColonyThemes() re-copies each launch.

- **Live theme preview on hover** — `Shell patch` · effort M · impact ●●●○○ · **72/100**
  Hover a Colony card to try the whole client on before committing.
  A MutationObserver reusing the .vc-settings-theme-grid hook colonyThemeHeaders.ts already owns attaches mouseenter/leave to each card; on hover it reads that theme's CSS via a new IpcEvents.readColonyTheme (handle() in ipc.ts reading DATA_DIR/themes/colony-*.css) and injects a <style id='colony-preview'>, removed on leave. Click still commits via Equicord's normal enable. Instant full-window preview without enabling, extending the exact grid SphereCord already augments.
  _Watch out:_ risk: preview style must outrank the enabled theme (later insertion + higher specificity) with careful teardown so a stray hover doesn't stick.

- **Colony-palette-aware splash screen** ⚡ — `Native` · effort S · impact ●●●○○ · **71/100**
  The loading screen wears whatever Colony theme you're running.
  Merges the two splash entries. splash.ts already supports splashTheming/splashColor/splashBackground; derive them at launch from the active Colony palette (bg_primary + accent) read from the same ~/.config/colony state the sync watcher uses (or by parsing the enabled colony-*.css), falling back to the default. Small change piggybacking on the theme-sync watcher that carries the palette into the shell's own chrome pre-Discord — Colony-native from the first frame. S-effort.
  _Watch out:_ risk: splash renders very early — read the palette synchronously with a hard-coded fallback so launch never blocks on a missing/locked config.

- **Per-server theme binding** — `Shell patch` · effort M · impact ●●●○○ · **68/100**
  Work servers go slate-Nord; the gaming server goes neon Synthwave.
  A renderer patch subscribes to FluxDispatcher CHANNEL_SELECT / guild changes (from @equicord/types/webpack/common), looks up a stored guildId→themeName map (Settings store), and injects the matching colony-*.css (via IpcEvents.readColonyTheme) into a <style id='colony-per-server'> only while that server is focused. Bind UI in the Settings tab or a guild context-menu item. Turns the bundled 52-palette set into per-context identity — the Colony-CSS binding is the differentiator vs generic per-server plugins.
  _Watch out:_ risk: rapid server hopping flashes — needs debounce + in-memory CSS cache.

- **Adaptive high-contrast + a11y (reduce-motion) mode** — `Colony` · effort M · impact ●●●○○ · **66/100**
  One Colony a11y setting, honored across the whole ecosystem including Discord.
  Merges 'Adaptive high-contrast toggle', 'High-contrast + language config sync', and 'Colony-synced accessibility mode'. Reads Colony's with_high_contrast() flag + a reduce-motion pref from ~/.config/colony and toggles two bundled CSS files (colony-a11y-contrast.css boosting text-layer var luminance per Colony's boost; colony-a11y-motion.css zeroing Discord transition/animation durations), installed via a copy-loop generalized from installColonyThemes(). A Settings toggle allows local override. Consumes Colony's existing a11y model as the single source of truth so Discord stops being the odd one out.
  _Watch out:_ risk: Discord has no honest reduce-motion path — motion CSS is a brittle animation-selector allowlist needing periodic maintenance; contrast ideally derives per-theme values, not one blanket overlay.

- **Custom font pack + layout density panel** — `Native` · effort M · impact ●●●○○ · **63/100**
  Pick a legibility-first font and tighten spacing — client-wide, one panel.
  Merges 'Custom font + layout density panel' + 'Accessibility font pack'. Ships fonts (Atkinson Hyperlegible, OpenDyslexic) under static/fonts/ + colony-font-*.css that @font-face-loads them and overrides --font-primary/--font-display, installed via the generalized static-asset copy-loop from startup.ts. A Settings section chooses font + a density scale (message padding/list-height/radius via overridable CSS vars) backed by a Settings store field; an optional native file picker (IpcEvents.pickFont) loads a local .ttf served through the vesktop:// assets protocol. Offline, update-proof, no CDN. Prototypes the shared radius/spacing tokens grounding suggests coordinating across Colony apps.
  _Watch out:_ risk: verify redistribution licensing (both fonts are permissive); Discord's font/spacing vars shift across the visual refresh and OpenDyslexic metrics can break tight layouts — conservative defaults.

- **Dynamic accent from OS / wallpaper** — `Native` · effort L · impact ●●●○○ · **55/100**
  Your desktop accent bleeds into Discord automatically.
  A native module reads the OS accent (GNOME/KDE via gsettings/dconf, or the dominant color of the current wallpaper), maps it to the nearest of Colony's 8 accents (or uses it raw), and feeds the same runtime accent-override <style> as the accent switcher via IPC. A Settings toggle picks 'follow OS accent' vs a fixed Colony accent. Best shipped as a bonus source feeding the accent switcher rather than standalone — a desktop capability Vencord can't touch, kept on-brand via Colony's accent vocabulary.
  _Watch out:_ risk: OS-accent sources fragmented across DEs/OSes; wallpaper sampling adds an image-decode dependency.

### Privacy & local-first hardening

- **Native telemetry & science firewall** — `Native` · effort M · impact ●●●●○ · **82/100**
  Kill Discord's tracking at the socket, not in a patch Discord can rewrite.
  A session.defaultSession.webRequest.onBeforeRequest filter in mainWindow.ts cancels requests to Discord's /api/*/science, /api/*/metrics, /error-reporting, sentry.io and other beacon hosts before they leave the process. Unlike Vencord's renderer-level NoTrack (patches known JS, re-fetched on Equicord update), a main-process block is authoritative, catches native-issued beacons, is update-proof, and ships on-by-default with a user-editable blocklist in Settings. The 'Colony privacy default profile' framing matches the ecosystem's opinionated-defaults philosophy.
  _Watch out:_ risk: over-blocking breaks features — needs a tested allowlist + escape toggle; partial overlap with NoTrack, value is the native/default/update-proof layer.

- **Screen-share privacy shield (auto-arm on capture)** — `Shell patch` · effort M · impact ●●●●○ · **80/100**
  The moment you go live, DMs, email, and invite links blur themselves.
  Merges 'Screen-share privacy guard' + 'Streamer Privacy Shield'. A renderer patch injects a body-level 'spherecord-streaming' class + CSS that blurs the DM/friends list, account-panel email/username, note-to-self, and pasted invite/token strings (blur lifts on hover); a MutationObserver scans rendered text for token/email/invite regexes. The killer bit is auto-arm: main/screenShare.ts knows when a capture starts and patches/tray.ts already subscribes to RTC_CONNECTION_STATE — reuse that Flux signal to toggle the class automatically, plus a manual Settings switch. Ties into the native screen-share lifecycle a pure-web plugin can't observe; extends Colony's existing venmic/screenshare work with a privacy layer.
  _Watch out:_ risk: regex sensitive-string detection has false neg (leaks) and false pos (over-blur) — blur is a local visual aid, never a security guarantee; notification-suppression must reliably restore on share end.

- **Default-deny web-permission & sensor hardening** ⚡ — `Native` · effort S · impact ●●●○○ · **78/100**
  Discord-in-a-browser can ask for USB and location. Say no, everywhere.
  Generalize the darwin-only mediaPermissions handler to all platforms: install session.setPermissionRequestHandler + setPermissionCheckHandler that deny geolocation, HID, serial, USB, MIDI, and web-notification by default (allowing only mic/camera/screen) and neutralize navigator.geolocation. A small native module registered from startup.ts with the allowed set surfaced in Settings. S-effort, secure-by-default matching Colony's zero-config ethos.
  _Watch out:_ risk: a future legit feature needing one of these would be silently blocked — keep the allowlist visible/editable.

- **WebRTC IP-leak guard + fingerprint consistency** ⚡ — `Native` · effort S · impact ●●●○○ · **76/100**
  Voice chat shouldn't hand strangers your home IP.
  In startup.ts init() append Chromium flags (force-webrtc-ip-handling-policy=default_public_interface_only, WebRtcHideLocalIpsWithMdns) so voice/screenshare don't expose local/public IPs via ICE candidates, aligned with gnuSpoofing.ts's existing UA/platform spoof so the fingerprint stays internally consistent. Pure flag/native work reusing existing spoofing infrastructure. S-effort, invisible, zero-config.
  _Watch out:_ risk: aggressive WebRTC policy can degrade P2P voice on some NATs — needs a fallback/off switch.

- **EXIF / metadata scrubber on upload** — `Shell patch` · effort M · impact ●●●●○ · **74/100**
  Your vacation photo shouldn't ship its GPS coordinates to a Discord CDN.
  A renderer patch intercepts the attachment pipeline (webpack-patch the upload/file-processing module, or wrap the paste/drop handler) and re-encodes images through an OffscreenCanvas before send, stripping EXIF/GPS/camera metadata — beyond Vencord's AnonymiseFileNames (which only renames), optionally randomizing the filename in the same pass. Toggleable per-session. Local-first data hygiene: nothing leaves the machine carrying more than intended.
  _Watch out:_ risk: canvas re-encode changes format/quality and can't cover video/PDF/other types — must warn or skip non-images.

- **Presence & read-receipt network shield** — `Native` · effort S · impact ●●●○○ · **70/100**
  Read the room without telling the room you're typing.
  A native webRequest rule in mainWindow.ts blocks outbound POST /channels/*/typing and message-ack endpoints so typing indicators and read receipts never leave the machine, toggleable in Settings. Overlaps Vencord SilentTyping conceptually, but the network layer makes it robust to Discord code changes AND covers acks (which SilentTyping doesn't). Ships default-off as a facet of the single Colony 'Private Mode' profile.
  _Watch out:_ risk: blocking acks can make unread counts behave oddly; typing half overlaps an existing plugin — value is the native + ack coverage.

- **Link-safety interstitial** — `Native` · effort M · impact ●●●○○ · **68/100**
  Catch the punycode phish before the browser ever opens.
  Hook the main-process link-out path (makeLinksOpenExternally / setWindowOpenHandler in mainWindow.ts) to run each outbound URL through a heuristic — punycode/homograph detection, known shortener/tracker-host lists, anchor-text-vs-href mismatch — and show a native confirm dialog with the decoded real destination before opening. Distinct from Vencord ClearURLs (silent param strip); this is a consent gate for deceptive links, extending the shell's existing link-routing responsibility.
  _Watch out:_ risk: bad heuristics cause dialog fatigue — needs per-domain 'always allow' memory + a tight default trigger set.

- **Local encrypted message vault** — `Colony` · effort L · impact ●●●○○ · **58/100**
  Your own words, backed up locally — not hostage to a server-side delete.
  A renderer FluxDispatcher subscription captures your own sent/starred messages (MESSAGE_CREATE where author == self) and ships them over IPC to a native module appending to an encrypted store under DATA_DIR, key held in Colony's OS keychain (reusing the oauth.rs keychain pattern). A local-first, offline-readable archive independent of Discord, browsable from the Settings tab. Ties the vault to Colony's existing secret store — but large scope and storing history is itself sensitive.
  _Watch out:_ risk: disk footprint + needs solid encryption and a wipe control; scope is large.

### Power-user QoL

- **Colony Command Palette (Ctrl+K)** 🚩 — `Shell patch` · effort L · impact ●●●●● · **87/100**
  One shortcut to rule Discord nav, SphereCord settings, and the whole Colony ecosystem.
  A fuzzy command-palette overlay as src/renderer/patches/commandPalette.tsx: a DOM observer mounts a fixed overlay + keydown listener (Ctrl/Cmd+K, or a global FIFO hotkey when unfocused) indexing three action sources — Discord nav (guilds/channels/DMs from GuildStore/ChannelStore, jump via NavigationRouter/transitionTo), SphereCord actions (toggle theme, open Settings tab, relaunch via VesktopNative), and Colony app launch (fire spherecord:// / colony:// deep-links so 'Open Grape'/'Open Spotter' work). Survives Equicord updates. Turns the palette into an ecosystem launcher — same muscle-memory jumps between a DM and another Colony app. Signature power-user surface; could host the keyboard theme switcher too.
  _Watch out:_ risk: Discord ships a Ctrl+K quick-switcher so the Colony/SphereCord action sources must be the clear differentiator; webpack store lookups (NavigationRouter, GuildStore) shift between builds and need defensive finds.

- **OS idle/lock/suspend-aware auto-status + auto-mute** — `Native` · effort M · impact ●●●●○ · **83/100**
  Lock your laptop and SphereCord goes Idle and mutes your mic automatically.
  Merges 'Desktop-Aware Auto Status' + 'OS idle/lock/suspend aware auto-status + auto-mute'. A native module hooks Electron powerMonitor ('lock-screen'/'unlock-screen', 'suspend'/'resume', getSystemIdleTime) and via a new VesktopNative IPC entry tells a thin renderer patch to flip Discord presence (Idle/Invisible) via a Flux dispatch and toggle self-mute via the existing TOGGLE_SELF_MUTE path. Reads REAL OS session state — works even when Discord is fully backgrounded, which Vencord's in-app idle polling fundamentally can't. Can piggyback the shared Colony focus flag.
  _Watch out:_ risk: the status setter/Flux dispatch can drift on Discord updates; lock events on Wayland/Linux vary by session manager.

- **Native message reminders & snooze** — `Native` · effort M · impact ●●●●○ · **80/100**
  Right-click a message, 'Remind me in 3h' — a real OS notification pulls you back.
  A renderer patch adds a 'Remind me' item to the message context menu capturing {messageId, channelId, guildId, fireAt}, handed to main via IPC and stored in State; at fire time main raises a native Electron Notification (reusing appBadge.ts plumbing) whose click deep-links back to the message and bumps the tray/app badge. Main-process backing means it fires even if the Discord tab is unloaded and survives restart — a guarantee a Vencord-only plugin can't make.
  _Watch out:_ risk: deep-linking to an exact message needs jump-URL reconstruction + channel transition; notification behavior differs across Linux/Wayland vs Windows.

- **Persistent message scheduler** — `Native` · effort M · impact ●●●●○ · **76/100**
  Queue a message now; the shell sends it at 9am even after a reboot.
  A composer clock-icon button (DOM observer on the channel textarea) opens a time picker; the pending {channelId, content, fireAt} goes to main over IpcEvents.SCHEDULE_MESSAGE, stored in main/settings.ts State. Main re-arms all pending timers on launch (startup.ts) so schedules survive restarts — the key desktop advantage over a web plugin — then signals the renderer to call MessageActions.sendMessage at fire time. Schedules could live under ~/.config/colony/spherecord for a shared source of truth.
  _Watch out:_ risk: sends on the user's behalf at a delayed time — guard against firing into a deleted channel or after logout, and keep the queued list clearly visible/cancellable.

- **Desktop jump-list / MRU for recent DMs** — `Native` · effort M · impact ●●●●○ · **74/100**
  Jump straight to a DM from the taskbar/tray without the window even focused.
  A native feature keeps a short MRU of channels/DMs (renderer reports recents over IPC as the user navigates) and publishes it to the OS: app.setJumpList (Windows taskbar) and extra items on the existing Linux StatusNotifierItem tray menu (tray.ts already builds custom menus). Selecting an entry focuses the window and deep-links to that conversation — an OS-level jump surface Vencord cannot touch, consistent with Colony's Linux-first native flavor.
  _Watch out:_ risk: Jump List is Windows-only; on Linux you rely on the tray submenu, so the feature's shape differs per-OS with graceful fallbacks.

- **Multi-account profile switcher (isolated sessions)** — `Native` · effort L · impact ●●●●○ · **72/100**
  Alt, work, and main accounts as separate tray-switchable windows — real isolation.
  A native feature using Electron session partitions (persist:profile-<id>) and per-profile DATA_DIR so each account has its own cookies/token/Vencord config; a tray submenu (extend tray.ts) lists profiles and opens/focuses a window per profile, CLI gains --profile <name>. Impossible for a renderer plugin — requires spawning BrowserWindows with distinct Electron sessions at the shell level. High demand feature for power users.
  _Watch out:_ risk: Discord ToS friction around multi-accounting; window/session lifecycle + arrpc/tray state must be made per-profile, touching several main modules.

## Gaps & gap-fillers (completeness pass)

What the first pass under-served:

- Localization is unused: Colony ships ~50-language i18n (i18n.rs) but SphereCord only reuses EN theme labels, so its own chrome (Settings tab, Colony family headers, tray menu) never follows the ecosystem's chosen language — a Colony asset the catalog ignores entirely.
- No onboarding / first-run experience: nothing adopts Colony's persisted theme/accent/language/identity on first launch, which clashes with Colony's zero-config value and leaves new users manually reproducing their ecosystem setup.
- Accessibility stops at high-contrast + reduce-motion: there is no colorblind validation/simulation of the 52 palettes, no text-scaling/dyslexia option, and no speech-to-text (only TTS output), leaving vision- and motor-impaired users underserved.
- Voice/audio power is thin: only screen-share/venmic fixes exist — no mic noise suppression, audio-device quick-switch, soundboard, or global push-to-talk despite the keybinds FIFO and venmic already being in-tree.
- No performance/resource awareness: Colony's Spotter is a GPU monitor, yet SphereCord surfaces no resource HUD, battery/low-power mode, or animation throttling — a natural Spotter tie-in and a fit for Colony's 'native performance' ethos.
- No data portability: the local-first theme covers an encrypted vault but nothing backs up or exports the SphereCord + Colony config/theme/asset setup for migration to a new machine.
- Content-creator / streaming needs are unmet beyond the capture-triggered screen-share shield: no persistent 'streamer mode' that redacts DMs, tokens, invites, and personal info while live.
- Ecosystem discovery is one-directional: SphereCord advertises Colony outward (presence, splash, badge) but offers no inward surface to browse/install/launch sibling Colony apps from inside Discord.
- Update/health UX is bare: the shell can download/repair the Equicord asar from the tray, but there is no rich update-status or diagnostics dashboard for users.
- Notification depth is missing scheduled Quiet Hours / DND despite OS-idle auto-status existing — no calendar/time-window-driven suppression.

## Suggested build order

Merged 76 raw ideas down to 48 surviving entries across 8 clusters. Heaviest dedupe: the 'Live Colony theme + accent sync watcher' appeared 4x (colony-integration/theming/native-desktop lenses) — collapsed into one flagship; 'runtime 8-way accent override' 3x; the two deep-link, two splash, two launcher-picker, two font, two mod-notes-adjacent, and three focus/DND/private-mode entries were each merged. Auto-status/auto-mute merged from two power-lens copies; presence hub merged with MPRIS/cross-app RPC framings but MPRIS bridge kept separate as the concrete Linux D-Bus implementation that feeds it.

Scoring favored shell-renderer-patch + native-main + colony-integration (all update-proof in this stack) over anything requiring an Equicord fork — no surviving idea needs a bundled Equicord plugin, so no fork-cost penalties applied. The theme-sync watcher (96) tops everything: highest colony-fit, reuses the existing 1:1 palette↔css mapping and installColonyThemes registration, and is the named signature seam. Several other Colony-integration ideas (Private Mode, splash, per-server themes) share its fs.watch(~/.config/colony) infrastructure — build the watcher first, then they become cheap add-ons.
